# FlashShield 演示操作手册

## 1. 一句话介绍

当 A 链仓位接近清算时，Reactive 会自动把风险事件带到 B 链，并在 B 链执行保护动作。

## 2. 推荐演示流程

1. 打开前端页面并连接钱包
2. 保持默认参数，或输入一个新的策略 ID
3. 点击 `1. 开启演示仓位`
4. 钱包第一次签名：在 A 链创建演示仓位
5. 点击 `2. 触发接近清算`
6. 钱包第二次签名：在 A 链发出风险事件
7. 页面播放价格滑落动画
8. Reactive 自动监听并把保护动作带到 B 链
9. 页面自动轮询并显示 B 链结果

## 3. 评委看到的重点

- A 链风险事件是真实链上事件
- Reactive callback 是真实跨链自动化
- B 链结果是真实链上状态，不是前端本地变量
- 当前 P0 版本采用固定保护参数：保护 80% 风险暴露

## 6. 前端当前行为说明

- `连接钱包`：只连接本页，不会偷偷自动连接
- `开启演示仓位`：需要第一次签名
- `触发接近清算`：需要第二次签名
- 第二次签名后不再需要用户手动确认，Reactive 和 B 链保护自动完成

## 7. 当前限制

- 当前 P0 保护比例固定为 80%
- 当前 Demo 主要验证跨链自动保护链路，不做真实 DEX 兑换
- 当前 B 链执行结果以策略是否真正触发为准，不再错误复用旧策略结果
