# FlashShield 部署说明

这份文档描述的是 **P0+ 动态对冲 demo** 的部署流程。  
当前阶段目标不是一次性完成全部真链路部署，而是先把部署参数、部署顺序和验收点固定下来。

## 1. 部署前准备

需要准备：

- Ethereum Sepolia RPC
- Base Sepolia RPC
- Reactive RPC
- 一个用于部署的测试钱包私钥
- 测试网 gas 余额
- Reactive callback 校验所需的 `Callback Proxy` 和 `RVM ID`

敏感信息只放在根目录 `.env` 中，不要提交到 Git。

可参考模板：

- [.env.example](../.env.example)

## 2. 部署顺序

### 第一步：部署 A 链风险模拟合约

部署目标：

- `PositionRiskSimulator`

部署后记录：

- `POSITION_RISK_SIMULATOR_ADDRESS`

### 第二步：部署 B 链保护执行合约

部署目标：

- `ProtectionExecutor`

部署时需要传入：

- `authorizedCallbackProxy`
- `expectedRvmId`
- `contractMultiplier`

部署后记录：

- `PROTECTION_EXECUTOR_ADDRESS`

### 第三步：部署 Reactive Contract

部署目标：

- `ReactiveProtection`

部署时需要传入：

- `originChainId`
- `destinationChainId`
- `originContract`
- `nearLiquidationTopic`
- `destinationExecutor`

部署后记录：

- `REACTIVE_PROTECTION_ADDRESS`

当前仓库已提供脚本骨架：

- [deploy-position-risk-simulator.js](../contracts/scripts/deploy-position-risk-simulator.js)
- [deploy-protection-executor.js](../contracts/scripts/deploy-protection-executor.js)

## 4. 部署后验收

至少检查以下内容：

1. A 链合约地址可读，能成功 `openPosition`
2. B 链合约地址可读，`contractMultiplier` 正确
3. Reactive 合约部署参数与 A/B 链地址一致
4. 前端配置已同步到最新合约地址

## 5. 真链路联调验收

联调时，按这条顺序检查：

1. A 链执行 `openPosition`
2. A 链执行 `updateMarkPrice`
3. A 链发出 `NearLiquidation`
4. Reactive 收到事件并发起 callback
5. B 链执行 `onReactiveCallback`
6. `ShortPositionOpened` 事件出现
7. 前端显示“ShortOpened / 已对冲”

## 6. 当前仍待你提供的参数

真正开始部署前，需要你提供：

- `ETHEREUM_SEPOLIA_RPC_URL`
- `BASE_SEPOLIA_RPC_URL`
- `REACTIVE_RPC_URL`
- `DEPLOYER_PRIVATE_KEY`
- `REACTIVE_CALLBACK_PROXY`
- `EXPECTED_RVM_ID`

如果 Reactive 侧还需要额外的特定账户、配置或网络参数，再在部署前补充。
