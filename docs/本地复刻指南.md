# FlashShield 本地复刻指南

这份文档面向第一次拿到仓库的人，目标是帮助你从 `git clone` 开始，在本地完成依赖安装、环境变量配置、合约测试、前端启动，以及最小演示链路验证。

---

## 1. 前置要求

建议使用以下环境：

- Ubuntu / WSL2
- Node.js 20.x
- npm 10.x
- 浏览器钱包，例如 MetaMask
- Ethereum Sepolia、Base Sepolia 测试 gas

当前项目已经在 WSL Ubuntu 环境下验证通过，不建议优先用 Windows 原生 Node 直接跑合约工具链。

---

## 2. 拉取仓库

```bash
git clone git@github.com:SeeMoon357/FlashShield.git
cd FlashShield
```

如果你使用 HTTPS：

```bash
git clone https://github.com/SeeMoon357/FlashShield.git
cd FlashShield
```

---

## 3. 安装依赖

### 合约侧

```bash
cd contracts
npm install
cd ..
```

### 前端侧

```bash
cd web
npm install
cd ..
```

---

## 4. 配置环境变量

先复制模板：

```bash
cp .env.example .env
```

然后在根目录 `.env` 中填写这些参数：

- `ETHEREUM_SEPOLIA_RPC_URL`
- `BASE_SEPOLIA_RPC_URL`
- `REACTIVE_RPC_URL`
- `REACTIVE_CHAIN_ID`
- `DEPLOYER_PRIVATE_KEY`
- `REACTIVE_CALLBACK_PROXY`
- `EXPECTED_RVM_ID`
- `POSITION_RISK_SIMULATOR_ADDRESS`
- `PROTECTION_EXECUTOR_ADDRESS`
- `REACTIVE_PROTECTION_ADDRESS`

注意：

- `.env` 不要提交到 Git
- 私钥只用于测试网
- 如果你准备复用当前仓库里已经部署好的地址，可以直接填写 README 中列出的部署地址

---

## 5. 运行合约测试

```bash
cd contracts
npx hardhat compile
npx hardhat test
```

如果你看到所有测试通过，说明本地 Solidity 工程是正常的。

## 7. 前端页面的正确使用顺序

页面上的主流程分为三段：

1. 开启演示仓位  
   这一步会在 A 链发起一次真实交易，需要第一次签名。

2. 触发接近清算  
   这一步会在 A 链发起第二次真实交易，需要第二次签名。

3. Reactive 自动回调到 B 链  
   这一步不再需要用户签名，系统会自动完成跨链保护。

---

## 8. 推荐的演示参数

建议第一次先用这组参数：

- 策略 ID：`FS-DEMO-01`
- 开仓价格：`100`
- 清算阈值：`88`
- 目标触发价：`90`

注意参数规则：

- `清算阈值` 必须小于 `开仓价格`
- `目标触发价` 应该低于 `开仓价格`

---

## 9. 如何验证链路是否真的跑通

你应看到以下现象：

### A 链侧

- 仓位成功创建
- 当前价格被推进到接近清算区间
- 页面显示已触发 `NearLiquidation`

### Reactive 侧

- 页面显示 Reactive 已接收到事件
- 不需要再进行第三次签名

### B 链侧

- 页面中的保护结果会自动刷新
- 当前策略成功时，风险暴露应从 `100 / 0` 变为 `20 / 80`

也就是：

- 风险资产：20%
- 稳定资产：80%
- 状态：已保护

---

## 10. 如果页面没有自动更新

可以先检查：

1. 当前策略 ID 是否是你刚刚开的那个
2. 是否已经等待了 10 到 20 秒
3. 钱包是否连接在正确网络
4. 根目录 `.env` 里的地址是否和当前部署记录一致

建议同时对照：

- [部署记录.md](./部署记录.md)
- [演示操作手册.md](./演示操作手册.md)
- [项目逻辑说明.md](./项目逻辑说明.md)

## 11. 当前版本的已知边界

本仓库当前是可提交的黑客松 Demo，而不是完整产品。当前边界包括：

- B 链保护动作是链上 mock 执行，不是真实 DEX swap
- 当前保护比例固定为 80%
- 还没有自动买回
- 还没有完整存入 / 取回资金池逻辑

这些边界不会影响“跨链自动保护链路已经成立”这个核心结论。
